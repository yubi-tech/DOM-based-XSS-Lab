About the Lab

This cybersecurity lab is designed to enhance your understanding of web application vulnerabilities. It focuses on practical, hands-on learning, enabling you to identify and exploit common security flaws.

Objectives

  • To familiarize participants with common web vulnerabilities.
  • To provide a safe environment for practicing penetration testing techniques.
  • To encourage ethical hacking and responsible disclosure of security issues.

How to Solve the Lab: DOM XSS Challenge

In this lab, you will encounter a DOM-based Cross-Site Scripting (XSS) vulnerability in the feedback page. Follow these steps to solve the challenge:

Steps to Solve the Lab

  1. Navigate to the Submit Feedback page.
  2. Observe the query parameter in the URL. It will look something like this: ?returnPath=/home.
  3. Right-click on the page and select Inspect or press Ctrl+Shift+I to open the developer tools.
  4. Locate the anchor element on the page. You can use the $ selector in the console to find it.
  5. Modify the query parameter returnPath to /randomString (replace randomString with any alphanumeric string).
  6. Change the returnPath value to javascript:alert(document.cookie).
  7. Press Enter and click the back link to trigger the alert.

Once you successfully execute the above steps, you will see an alert displaying the cookies, confirming that you have exploited the DOM XSS vulnerability!
